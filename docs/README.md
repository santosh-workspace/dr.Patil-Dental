# Dr. Ghule's Dental Clinic — Website

Production website generated by the **Dental Website Design System**.
Stack: **Next.js (App Router) · TypeScript · Tailwind CSS · Framer Motion**.

---

## Quick start

```bash
npm install
npm run dev      # http://localhost:3000
npm run build    # production build
npm start        # serve the production build
```

> **Note:** `app/layout.tsx` uses `next/font/google` (Plus Jakarta Sans + Manrope).
> The build fetches these fonts once at build time, so an internet connection is
> required for `npm run build`. Everything else builds and runs offline.

---

## What's implemented

**Theme:** Professional Blue · **Header:** Classic Medical · **Hero:** Classic Split ·
**CTA:** Centered · **Footer:** Classic · **Fonts:** Plus Jakarta Sans + Manrope.

- Mobile-first, fully responsive, WCAG 2.2 AA (skip link, focus states, ARIA,
  keyboard-operable nav/accordion/forms, reduced-motion support).
- Token-driven design — colours, type scale, radius and shadow are all CSS
  variables (`styles/tokens.css` + `tailwind.config.ts`). Swap the theme by
  editing tokens only; no component changes required.
- SEO: per-page metadata, canonical, Open Graph, Twitter, `sitemap.xml`,
  `robots.txt`, semantic headings (single H1 per page).
- Local SEO + Schema (JSON-LD): `Dentist`/LocalBusiness, `Organization`,
  `WebSite`, `FAQPage`, `BreadcrumbList`, service `ItemList`, opening hours,
  address, and `areaServed`.
- Framer Motion: fade / reveal / scale / stagger presets, reduced-motion safe.
- Pages: Home, About, Services (with per-service anchors), Doctor, Gallery,
  FAQ, Contact, Privacy Policy, plus 404 / error / loading states.
- Working Appointment + Contact forms (React Hook Form + Zod) that open a
  pre-filled WhatsApp message to the clinic. Wire to an email/API service when ready.

## All client data is real

Clinic name, doctor, qualification, registration, experience, phone/WhatsApp,
email, address, hours, Google Maps/Business Profile link and doctor photo are
taken verbatim from the client. **Nothing was fabricated.**

## Open items (need the client)

1. **Clinic & gallery photos** — The Technology and Gallery sections now use
   free stock photography from Pexels (see `data/media.ts`). These are generic,
   decorative dental photos representing the type of care provided — **not photos
   of this specific clinic**. For full authenticity, replace them with real
   clinic photography: drop files into `public/images/` and update the `src`
   values in `data/media.ts`. Each image has an illustration fallback, so the UI
   never breaks. Photographer credits (Pexels license) are recorded in
   `data/media.ts`; self-hosting the images is recommended for production.
2. **Patient reviews** — The Reviews section is a premium testimonial layout
   (rating summary, Google branding, animated counters, stagger reveal, hover
   elevation). It currently shows clearly-labeled **sample** reviews from
   `data/sample-reviews.ts` for design preview, using generated illustrated
   avatars (DiceBear — not real people's faces) and **no review schema**. Add
   genuine reviews to `data/reviews.ts` to replace the samples and switch on
   `AggregateRating` + `Review` schema. Only real, verifiable reviews should be
   published — fabricated testimonials
   are prohibited by the Content Rules and by consumer-protection law.
3. **Logo** — a text wordmark + tooth mark is used by default. Drop a logo into
   `public/logos/` and update `components/shared/Logo.tsx` if a graphic logo exists.
4. **Live domain** — set the real domain in `constants/site.ts` (`site.url`);
   it currently uses a placeholder. This drives canonical URLs, sitemap and schema.
5. **Open Graph image** — add `public/images/og-default.jpg` (1200×630) for
   richer social sharing.
6. **Google Maps vs Business Profile** — the same `share.google` link was supplied
   for both and is used for both. Replace with a distinct Business Profile link if one exists.
7. **Form delivery** — forms open WhatsApp; connect to email/CRM if preferred.

## Structure

```
app/            routes, layout, sitemap, robots, error/loading/not-found
components/     animations · cards · forms · layout · navigation · sections · shared · ui
constants/      site (single source of clinic data) · routes
data/           services · faq · why-choose-us · technology · stats
lib/            utils · metadata · schema · validators · maps
animations/     Framer Motion variant presets
hooks/          reusable client hooks
styles/         design tokens (CSS variables)
types/          shared TypeScript types
public/         images · logos
```
